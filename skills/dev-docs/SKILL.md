本项目采取极简文档策略。文档的目标是给新加入开发的 Agent 一个 Quick Start，而不是把代码逻辑在文档中重述一遍。

我们管理以下文档文件：

## {project_root}/AGENTS.md

自动加载到 Agent 上下文的项目指引。
要求：有且只有 essential 信息。
格式如下：

```
# {project_name} 项目指引

## 目标（项目核心目标和视野）

## 模块地图（主要模块的地图，用于让 Agent 快速了解项目概况和各模块职责。注意：保持精简。这不是全部文件的地图也不是全部目录的地图。）

## 开发与测试（开发者须知的各类命令、规范、流程）
```

## {project_root}/docs/features

需求开发记录，包括计划、实施总结。在需求开发过程中实时更新。当需要回顾特定需求的策略和进展时再查看。

## {project_root}/docs/research

调研报告。仅在用户明确要求时落盘。
